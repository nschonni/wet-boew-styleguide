Description

By default, grids cells float left and stack side-by-side, forming rows from left to right. However, you can use positional CSS to layer them on top of each other. The parent cell will act as the container, and the child cells can be layered horizontally and vertically within the container, on top of other cells.

Note: The layer that is to appear behind the absolutely position object must have a CSS property that is anything but static. For example, grids have a CSS property of position: relative, so they work well with this design feature.

Positioning options

The broswer will always position content on the top left. To modify the default placement, both the x and y position must be defined. The x positions are pstn-left-* and pstn-rght-*. The y positions are pstn-tp-* and pstn-bttm-*. Since positions and work together, then the posting is also responsive, so that when the grid system linearizes for smaller viewports, so do the positioning.

Top positioning Right positioning Bottom positioning Left positioning

.pstn-tp-xs = always positioned

.pstn-tp-sm = tablets upwards only

.pstn-tp-md = small desktop upwards only

.pstn-tp-lg = large desktop upwards only

.pstn-rght-xs = always positioned

.pstn-rght-sm = tablets upwards only

.pstn-rght-md = small desktop upwards only

.pstn-rght-lg = large desktop upwards only

.pstn-bttm-xs = always positioned

.pstn-bttm-sm = tablets upwards only

.pstn-bttm-md = small desktop upwards only

.pstn-bttm-lg = large desktop upwards only

.pstn-lft-xs = always positioned

.pstn-lft-sm = tablets upwards only

.pstn-lft-md = small desktop upwards only

.pstn-lft-lg = large desktop upwards only

Adjust the margin

Content is always positioned flush with the edges. To add a margin, use the margin proxmity classes.
